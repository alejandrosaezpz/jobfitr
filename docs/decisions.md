# Decisions

## D-001 — Documentation-first skeleton

- **Status:** Accepted
- **Decision:** Build only the repository structure and documentation base first.
- **Rationale:** Reduce ambiguity before introducing code.

## D-002 — No functional backend/frontend yet

- **Status:** Accepted
- **Decision:** Do not add runtime application logic at this stage.
- **Rationale:** The approved scope is structural and documentary only.

## D-003 — Fictitious examples allowed

- **Status:** Accepted
- **Decision:** Use illustrative examples where helpful, clearly marked as fictitious.
- **Rationale:** Support orientation without implying implementation.
