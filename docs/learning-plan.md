# Learning Plan

## Objective

Use this repository as a controlled starting point for future implementation work.

## Suggested learning order

1. Read `README.md`
2. Review `docs/glossary.md`
3. Review `docs/decisions.md`
4. Confirm the roadmap in `docs/roadmap.md`
5. Implement only the next approved layer

## What to avoid

- Adding framework code before the stack is approved
- Creating features without documented intent
- Mixing placeholders with production assumptions
